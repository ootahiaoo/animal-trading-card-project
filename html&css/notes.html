HTML class notes

HTTP (Hypertext Transfer Protocol): rules for how requests and responses work (tells the browser how to talk to the server)
HTTPS (Hypertext Transfer Protocol Secure)

HTML (Hypertext Markup Language) is made of Text, Markup and References.

  • Markup : tells the browser what the text should look like or how it's arranged
  • References : used to include/link to other documents and files
  • Whitespace : spaces, tabs, line breaks

Void element (empty element) : HTML element with no context & no closing tag

Headlines/Section headings use the following tags:

  1. h1 (biggest size)
  2. h2
  3. h3
  4. h4
  5. h5
  6. h6 (smallest size)

Block elements form a view, while Inline elements don't.

Lists

  • The lists either ordered or unordered.
    1. ordered lists start with a number
    2. unordered lists start with dots or bullets
  • The tag inside doesn't need a closing tag.
  • The lists can be nested.

Links

URL: Uniform Resource Locator

  • When using the a tag, must use a fully qualified url (or absolute url)
  • A stands for anchor
  • The attribute of the anchor tag is href , which stands for hypertext reference.

Example: Github

Images

  • Needs 2 attributes
    1. src (source), the address of the image
    2. alt (alternate text), the text displayed if the user can't see the image or the image is not displayed
  • Can use Relative URL if the file is in the same local directory (URLs with no "://" will be treated as relative URL)
  • Is an empty element
  • Is an inline element

HTML page structure

  • Needs a DOCTYPE tag at the top of the page (if there isn't, the browser will use quircks mode to try to be compatible with older HTML versions)
  • Should specify the head and body (although it can be guessed)
    • Inside the head: the title, style, script and other metadata
    • Inside the body: the actual contents of the page
  • Best to specify the language used in the html tag (for accessibility)
